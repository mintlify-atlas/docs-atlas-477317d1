## Overview

After placing an order, you can view detailed order information, track payment status, and access pickup details at `/buyer/order/{id}`. This page provides everything you need to complete your purchase and pick up your items.

## Payment Status Types

Orders can have three different payment statuses:

### PENDING

Venmo payment initiated but not yet verified:

```typescript
export const BasicOrderSchema = z.object({
  id: z.string().uuid(),
  paymentMethod: z.enum(["VENMO", "OTHER"]),
  paymentStatus: z.enum(["UNVERIFIABLE", "PENDING", "CONFIRMED"]),
  pickedUp: z.boolean(),
  createdAt: z.coerce.date(),
  updatedAt: z.coerce.date(),
  // ...
});
```

Source: `common/schemas/order.ts:6-16`

<Warning>
**PENDING** status means:
- You've submitted the order
- Payment has not been verified yet
- For Venmo orders: The system is waiting to detect your payment
- **Action Required**: Complete Venmo payment if you haven't already
</Warning>

### CONFIRMED

Payment successfully verified:

```typescript
if (order.paymentStatus === "CONFIRMED") {
  return {
    borderColor: "border-green-500",
    textColor: "text-green-800 dark:text-green-200",
    title: "Payment Confirmed",
    message: "Your payment has been verified. No further action is required.",
  };
}
```

Source: `frontend/src/app/buyer/order/[id]/page.tsx:82-89`

<Note>
**CONFIRMED** status means:
- Payment has been verified
- Order is ready for pickup
- No further action required from buyer
</Note>

### UNVERIFIABLE

Payment cannot be automatically verified:

```typescript
// For OTHER payment method OR unverifiable VENMO
if (order.paymentMethod === "OTHER" || 
    (order.paymentStatus === "UNVERIFIABLE" && order.paymentMethod === "VENMO")) {
  return {
    borderColor: "border-blue-500",
    textColor: "text-blue-800 dark:text-blue-200",
    title: "Order Processed",
    message: "Your order has been received. Payment will have to be manually verified by the fundraiser managers.",
  };
}
```

Source: `frontend/src/app/buyer/order/[id]/page.tsx:93-115`

<Card title="UNVERIFIABLE Scenarios" icon="alert-circle">
  Orders receive UNVERIFIABLE status when:
  - Payment method is "OTHER" (cash in-person)
  - Venmo payment couldn't be automatically matched to the order
  - Manual verification by fundraiser managers is required
</Card>

  <Step title="Auto-filled Payment Details">

## Server-Side Data Fetching

The order page requires authentication and fetches data server-side:

```typescript
export default async function OrderPage({
  params,
}: {
  params: Promise<{ id: string }>;
}) {
  await connection(); // dynamic rendering
  
  const supabase = await createClient();
  const id = (await params).id;
  
  // Protect page - require authentication
  const { data: { user }, error: error1 } = await supabase.auth.getUser();
  if (error1 || !user) {
    redirect("/");
  }
  
  // Get JWT token
  const { data: { session }, error: error2 } = await supabase.auth.getSession();
  if (error2 || !session?.access_token) {
    throw new Error("No session found");
  }
  
  // Fetch order data
  const order = await serverFetch(`/order/${id}`, {
    token: session.access_token,
    schema: CompleteOrderSchema,
  });
  
  const fundraiser = await serverFetch(`/fundraiser/${order.fundraiser.id}`, {
    token: session.access_token,
    schema: BasicFundraiserSchema,
  });
  // ...
}
```

Source: `frontend/src/app/buyer/order/[id]/page.tsx:31-66`

<Warning>
Orders are protected routes. Unauthenticated users are redirected to the home page. Additionally, users can only view their own orders (enforced by the backend).
</Warning>
